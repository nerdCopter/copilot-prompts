# Markdown Efficiency Standards

- **Brevity:** Keep documentation minimal; avoid redundant files or verbose explanations.
- **Navigation:** If headers ≥ 5, provide a Table of Contents (TOC).
- **Anchors:** Format: `[Text](#lowercase-kebab-case-header)`.
- **Numbering:** Reserve `#XXXX` for GitHub issue references only. Use `No. 1`, `Number 1`, `Issue A`, or `Item # 1` (with spacing) for local documentation lists and items.
